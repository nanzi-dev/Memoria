# Memoria 开发路线图

## 已完成功能

### 第一阶段 - 核心对话系统
- [x] 结构化 JSON 角色卡定义（Pydantic 校验，11 个嵌套模型）
- [x] 角色卡加载器（LRU 缓存，数据库优先 / 文件回退，热重载）
- [x] 短期记忆（会话窗口，默认 8 轮）
- [x] 长期记忆（自动萃取 + SQLite 存储 + 文本相似度去重）
- [x] 关系与情感系统（好感度 -100~100、信任度、情绪实时追踪）
- [x] LLM 调用适配层（OpenAI 兼容接口，支持 DeepSeek/Kimi/Qwen）
- [x] Prompt 组装器（角色设定 + 记忆上下文 + 状态注入）
- [x] 单角色对话编排器 (Orchestrator)
- [x] 沉浸感保护（AI 身份检测、角色一致性约束）
- [x] 三层容错机制（JSON 解析 → 修复重试 → 文本兜底）
- [x] FastAPI RESTful API + 自动生成 OpenAPI 文档

### 第二阶段 - 增强记忆与管理
- [x] RAG 向量检索（ChromaDB + all-MiniLM-L6-v2，余弦相似度）
- [x] 向量记忆自动同步（插入时同步到 ChromaDB）
- [x] 中期记忆（会话摘要，AI 自动生成，同会话去重 upsert）
- [x] **记忆去重引擎** — `_text_similarity`（SequenceMatcher）+ `_dedup_check`（LIKE 粗筛→精确比较），覆盖 long_term_fact、session_summary、shared_memory、group_memory 四张表，阈值 0.75
- [x] 角色卡管理后台 API（CRUD + 导入导出 + 软删除 + 激活/禁用）
- [x] SQLite WAL 模式（支持并发读写）

### 第三阶段 - 事件系统
- [x] 事件数据模型（TriggerCondition / EventEffect / EventDefinition，Pydantic）
- [x] 事件检测引擎（14 种触发枚举 + 复合条件 AND/OR；`item_acquired`、`quest_completed`、`relationship_change` 检测尚未实现）
- [x] 事件执行器（状态修改、内容解锁、对话触发、记忆添加、情绪改变、玩家通知、事件链、分支事件、主动发言和事件进度更新；`grant_item`、`start_quest`、`modify_relationship` 枚举已保留但执行尚未实现）
- [x] 事件冷却时间管理（按 character+player 记录最后触发时间）
- [x] 事件触发日志（完整上下文快照 + 效果列表）
- [x] 角色关系网络 CRUD API（无向关系，自动排序唯一约束）
- [x] 事件管理 API（CRUD + 启用/禁用 + 触发历史查询 + 重置）

### 第四阶段 - 多角色对话
- [x] 多角色群聊（至少 2 个不重复 NPC；公开 API 未设置参与人数上限，MultiCharacterOrchestrator）
- [x] 固定混合发言策略（MultiCharacterOrchestrator 始终使用 HybridStrategy；其他策略类未作为公开选项暴露）
- [x] 讨论模式（角色连续发言，每轮按上下文动态选择回应人数；玩家发言后最多 3 个角色回应）
- [x] 混合策略评分（关键词、角色关系、发言频率、近期活跃度等信号采用加法评分，并结合随机选择；不是固定百分比权重）
- [x] 角色间互动（主动发言触发 + 对话历史上下文注入）
- [x] 三层多角色记忆 — long_term_fact（个人）+ shared_memory（角色间）+ group_memory（群体），含 DDL、CRUD、去重
- [x] 参与者状态与发言频率持久化（不包含运行中动态增删参与者的完整管理能力）
- [x] `group_thread_id` 逻辑群聊线程（跨物理 session 聚合历史，结束 session 后可建立新的 active carrier session 续聊）
- [x] 普通/事件自主脉冲（租约防重；当前每次最多 1 条消息；事件脉冲绕过普通冷却和每日预算）
- [x] 群聊脉冲原子提交（消息、关系、参与者、线程状态和未读通知同事务提交；记忆提取在提交后 best effort 执行）
- [x] 群聊结束整场摘要记忆（有效消息数大于 6 且摘要非空时写入 session_summary 和 group_memory）
- [x] 关系图谱权威链路（关系创建/更新/删除记录修订时间；单聊和多角色上下文只过滤图谱变更前的旧关系事实，结束摘要过滤旧关系历史）

## 已完成阶段与后续规划

### 第五阶段 - 质量与稳定性 ✅

**目标：** 提升系统健壮性，为生产环境做准备。

**当前进度：** 测试集合持续扩展；记忆去重、重试、对话轮次幂等、认证限流、健康检查、懒加载、会话恢复、用户资源隔离、关系图谱修订过滤、知识文档恢复、PostgreSQL 兼容和 API/安全回归测试已上线。

- [x] 单元测试覆盖（覆盖 schema、repository、编排器、事件、API、安全、知识库、语音、向量存储、世界时钟、开发者体验和 PostgreSQL 兼容；实时数量以 `pytest --collect-only -q` 为准）
- [x] 单聊/群聊轮次幂等（客户端 `request_id`、租约、完成结果和错误持久化，安全处理重复提交）
- [x] 单聊/群聊 SSE 流式输出（保留传统请求路径作为前端回退）
- [x] 群聊增量历史与未读通知（支持按 `after_message_id` 拉取及逻辑线程标记已读）
- [x] 持久后台 checkpoint memory（单聊/群聊任务、租约、去重、重试和终态持久化）
- [x] 数据库迁移到 PostgreSQL（保留 SQLite 开发模式）
- [x] LLM 调用重试机制（指数退避 3 次，base_delay=1s，max 7s）
- [x] 请求速率限制（60s 窗口 60 次；认证用户优先、未登录回退 IP；线程安全、单调时钟和过期 key 清理；额度为单进程）
- [x] 结构化日志（LOG_LEVEL 环境变量控制）+ 动态调整（POST /admin/log-level）
- [x] 健康检查端点（`/health` 存活 + `/ready` 数据库就绪）
- [x] 配置文件校验（启动时检查 LLM_API_KEY 等必需项，警告而非崩溃）
- [x] 内存使用优化（LLM Client 懒加载 + vector_memory 懒加载，首次调用时初始化）
- [x] CI/CD 流水线（GitHub Actions: Python 3.10/3.11/3.12 自动测试）

### 第六阶段 - Web 前端 ✅

**目标：** 提供开箱即用的 Web UI，降低使用门槛。

- [x] 单角色对话界面（角色头像、消息气泡、会话列表、会话恢复）
- [x] 多角色群聊界面（已集成在 ChatRoom，支持创建时选择参与者、恢复、逻辑线程历史加载和未读状态）
- [x] 角色卡编辑器（基础角色卡分步表单：身份→性格→语言风格→背景→交互规则；不提供 i18n 版本切换，保留已有覆盖数据）
- [x] 玩家角色卡（独立页面编辑玩家名称、头像、身份、外观、性格、背景和目标，并进入对话 Prompt 与关系图）
- [x] 事件管理工作台（汇总、搜索、筛选、排序、列表/详情分栏和已有角色选择）
- [x] 角色关系图谱（D3 力导引图）
- [x] 当前会话关系变化展示（新回复显示好感度/信任度 delta，历史加载消息不重复显示）
- [x] 用户登录/注册与资料设置（含用户头像）
- [x] 响应式设计（桌面端与移动端布局）
- [x] 知识库管理工作台（汇总、搜索、筛选、列表/详情分栏、创建/绑定、文档上传或粘贴、状态轮询、失败重试、检索预览）
- [x] Cookie-only 浏览器登录态（HttpOnly Cookie，不持久化 Bearer token）
- [x] 中英会话切换与语音交互（录音转写、自动发送、角色语音播放）
- [x] 用户设置分区与世界时钟（账户/世界时间/语音，IANA 时区、暂停、同步、设置、推进和 `0/1/2/5/10` 倍速）

### 第七阶段 - 事件系统深度集成

**目标：** 让事件系统真正影响对话体验，而非孤立的数据模型。

- [x] 事件与对话编排器集成（对话轮次后自动触发检测 + 执行）
- [x] 事件链（一个事件触发另一个事件，支持分支）
- [x] 时间驱动事件（定时检查 + cron 式调度）
- [x] 事件模板库（预设常见 RPG 事件：好感度里程碑、关键剧情节点）
- [x] 事件效果中的 NPC 主动对话（通过多角色编排器发起）
- [x] 事件上下文持久化（跨会话保留事件进度）

### 第八阶段 - 开发者体验

**目标：** 让二次开发和调试更高效。

- [x] CLI 调试模式（`--debug` 标志：显示 LLM 原始请求/响应、Prompt 内容）
- [x] 对话回放工具（加载历史 session，逐步重放查看状态变化）
- [x] 性能分析端点（LLM 调用耗时分布、记忆检索耗时）
- [x] Docker 一键部署（`docker compose up`）
- [x] 可选 OpenTelemetry 追踪（安装并启用相关依赖时记录 LLM 调用链路和数据库查询）
- [x] 对话质量评分 API（角色一致性、趣味性自动评估）

### 第九阶段 - 高级特性

**目标：** 扩展系统能力边界。

- [x] 知识库注入（外部世界观文档、角色/群聊绑定、单聊/群聊 RAG 检索、来源追溯）
- [x] 语音合成集成（TTS、磁盘缓存、内置声音和角色 Custom Voice 工作流）
- [x] 语音识别集成（STT，按会话语言转写，语音输入 → 文本 → LLM）
- [x] 双语支持（`zh-CN` / `en-US`，角色卡 i18n 覆盖、Prompt 与 STT 语言切换）

### 第十阶段 - 生态与发布

**目标：** 构建社区和分发渠道。

- [ ] 插件系统（自定义事件处理器、记忆存储后端、LLM Provider）
- [ ] 角色卡市场（上传/下载/评分/评论）
- [ ] 运行中动态参与者管理（后端完整增删流程及 Web UI）
- [ ] WebSocket 后台状态推送（仅用于非请求式通知等后台状态；请求式对话流已使用 SSE）
- [ ] 导出对话记录（Markdown / PDF / HTML）
- [ ] SDK 发布（Python `pip install memoria` + TypeScript 客户端库）
