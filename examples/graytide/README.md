# 灰潮港：第十三声钟鸣

这是 Memoria 的独立演示与压力测试模块。故事发生在一座依靠以太机械抵御周期性灰潮的港城：午夜后，全城听见主钟不可能敲出的第十三声，随后第七码头从所有公开航图上消失。玩家加入联合调查组，在互相矛盾的证词、受损自动机记忆和被涂改的工业账册之间建立证据链。

模块包含：

- 8 张角色卡，覆盖调查者、走私船长、档案员、机械师、医生、议会代表、信使和自动机。
- 18 条有正有负的角色关系。
- 24 个事件，其中 20 个启用，4 个作为未实现扩展或边界测试夹具保持禁用。
- 4 个知识库和 8 篇 Markdown 文档，覆盖全局、角色和群聊线程绑定。
- 1 个固定群聊线程与 10 条检索评测问题。

逐步调查路线和可直接使用的提问见 [WALKTHROUGH.md](WALKTHROUGH.md)。

## 播种

首次创建演示账户时必须通过参数或环境变量提供密码：

```bash
python scripts/seed_graytide_demo.py --password 'your-local-password'
```

也可以设置 `MEMORIA_DEMO_PASSWORD`。账户已经存在时不会修改密码。默认会使用项目配置的本地向量模型建立知识索引；只需要结构数据时可运行：

```bash
python scripts/seed_graytide_demo.py --skip-knowledge-index
```

需要重建本模块时使用 `--reset-module`。该选项仅清理 `memoria_demo` 名下、由本目录清单明确列出的灰潮港数据，然后立即重新播种。

## 调查起点

推荐在“第十三声钟鸣联合调查组”中从以下问题开始：

1. “请按时间顺序复盘钟鸣前半小时发生的事。”
2. “阿特拉斯的记忆为什么会比钟声更早被封锁？”
3. “零号货单、蓝色冷凝液和逆向棘轮有什么共同来源？”
4. “谁有动机掩盖第七码头仍在运作？”

内容刻意保留多个可疑对象和尚未闭合的证据环，避免系统只靠单一关键词直接给出答案。
