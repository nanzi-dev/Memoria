# 灰潮港：第十三声钟鸣

这是 Memoria 的独立演示与压力测试模块。故事发生在一座依靠以太机械抵御周期性灰潮的港城：午夜后，全城听见主钟不可能敲出的第十三声，随后第七码头从所有公开航图上消失。

玩家以“岑澜”的身份加入调查。岑澜是参与过第七共振支路复测的独立潮痕测绘员，持有半枚测绘牌和一张未被改写的蓝线航图。玩家需要利用自己的专业知识连接地图、压力路线与证物保管链，同时决定相信谁、公开什么，以及何时拒绝过早定罪。

模块包含：

- 1 张通过现有用户角色卡功能播种的玩家卡。
- 8 张 NPC 角色卡，覆盖调查者、走私船长、档案员、机械师、医生、议会代表、信使和自动机。
- 26 条有正有负的关系，其中 8 条直接连接玩家与全部 NPC。
- 24 个事件，其中 20 个启用，4 个作为未实现扩展或边界测试夹具保持禁用。
- 4 个知识库和 8 篇 Markdown 文档，覆盖全局、角色和群聊线程绑定。
- 1 个固定群聊线程与 12 条检索评测问题。

逐步调查路线和可直接使用的提问见 [WALKTHROUGH.md](WALKTHROUGH.md)。

## 播种

首次创建演示账户时必须通过参数或环境变量提供密码：

```bash
python scripts/seed_graytide_demo.py --password '<choose-a-strong-password>'
```

也可以先设置 `MEMORIA_DEMO_PASSWORD`，再省略 `--password`。账户已经存在时不会修改密码。默认会使用项目配置的本地向量模型建立知识索引；只需要结构数据时可运行：

```bash
python scripts/seed_graytide_demo.py --skip-knowledge-index
```

需要重建本模块时使用 `--reset-module`。该选项仅清理 `memoria_demo` 名下、由本目录清单明确列出的灰潮港数据，然后立即重新播种。

播种后可在用户角色卡页面看到“岑澜”。该身份会进入单人对话、群聊和关系图提示词；重新运行播种脚本会把演示账户的玩家卡恢复为本模块定义。

`memoria_demo` 始终是普通用户，播种脚本不会提交 `ADMIN_BOOTSTRAP_TOKEN`，也不会创建或占用系统管理员初始化名额。

模块结构、幂等播种、隔离清理和评测数据可通过以下测试验证：

```bash
.venv/bin/pytest -q tests/test_graytide_demo.py tests/test_story_module.py
```

## 调查起点

推荐在“第十三声钟鸣联合调查组”中从以下问题开始：

1. “艾琳，为什么需要我这名潮痕测绘员加入？我的半枚测绘牌和蓝线航图能证明什么？”
2. “请按时间顺序复盘钟鸣前半小时发生的事。”
3. “阿特拉斯的记忆为什么会比钟声更早被封锁？”
4. “零号货单、蓝色冷凝液和逆向棘轮有什么共同来源？”
5. “哪些证据指向批准链，哪些证据仍不能指认实际执行者？”

内容刻意保留多个可疑对象和尚未闭合的证据环，避免系统只靠单一关键词直接给出答案。
