# 回声档案

《回声档案》是 Memoria 的原创七卷现代都市调查模块。故事发生在河海交界的虚构城市临岚市，玩家固定扮演男性犯罪心理顾问许观澜，与刑警季衡及跨机构团队处理六宗可独立结案的单位案件，并在终卷处置贯穿全城的“闭环”档案。

本模块只保留现代城市调查、单元卷、群像关系与长线阴谋等高层类型特征。人物、机构、案件、对话、关系和叙事细节均为原创，不复用、改写、影射或模仿任何既有小说的角色、案件、标志性场景、台词、关系动态或文风。成人犯罪主题采用克制、非猎奇写法。

## 播种

从仓库根目录运行：

```bash
python scripts/seed_story_module.py examples/echo_archive \
  --password '<choose-a-strong-password>'
```

首次创建 `nanzi` 账户时必须提供 `--password`，也可以设置 `MEMORIA_DEMO_PASSWORD` 后省略该参数。账户已经存在时不会修改已有密码；命令示例只使用密码占位符，不应把真实密码写入仓库。

可选参数：

- `--skip-knowledge-index`：创建知识库和文档，但跳过本地知识向量索引。
- `--reset-module`：清理并重建当前演示用户下由清单声明的模块内容；每个声明逻辑线程中现有的最新物理 session 会被结束并复用，但既有逻辑线程不会被删除。

播种脚本会创建玩家卡、22 张 NPC 角色卡、关系、7 个固定卷组、事件、知识库和知识文档。模块内容不包含现实机构数据或现实违法操作说明。

## 群聊线程与脉冲

播种时，清单声明的群聊会被结束并按逻辑线程复用。某个线程首次播种时，固定的 `echo_vN_session` 会以 `ended` 状态创建；进入对应群聊后，可在界面点击“继续”，或调用：

```text
POST /api/v1/multi-dialogue/session/{session_id}/continue
```

继续操作会创建或复用一个 `active` 物理 session。一个卷组前后产生的多个物理 session 共享同一个 `group_thread_id`，因此历史和未读状态都按逻辑线程聚合。可用以下接口分页同步完整线程：

```text
GET /api/v1/multi-dialogue/history/{session_id}
```

完整同步后，可调用以下接口清除该线程的聚合未读：

```text
POST /api/v1/multi-dialogue/thread/{group_thread_id}/read
```

离开界面期间，群聊可能产生后台脉冲。普通脉冲只有在世界时钟运行、距离上次脉冲至少经过 2 分钟真实时间和 20 分钟世界时间、当日 24 条自主消息预算尚未用尽、至少有 2 名活跃参与者，并且存在未解决剧情钩子、角色目标或可自然延续的话题等剧情动机时才会执行。事件脉冲绕过普通脉冲的冷却和预算限制；当前每个脉冲最多生成 1 条消息。

事件历史推进与群聊脉冲对白是不同概念：`EVENT_HISTORY` 依赖读取已经提交的事件历史，后台出现一条脉冲对白并不等于对应事件前置已经提交。
