
## 1. 产品隐喻与设计系统

- **核心概念**：“微信遇上赛博朋克 RPG”。高信息密度，保留熟悉的聊天流，但在视觉上呈现未来科技感和游戏质感。
    
- **主题与视觉规范（Design Tokens）**：
    
    - **卡片样式**：毛玻璃效果（`backdrop-blur-md bg-slate-900/60 border border-slate-800`）。
        
    - **视觉元素**：细致的 HUD 边框、科技感方括号 `[ ]`、微交互动画、状态改变时的数字故障（Glitch）效果、数据流般的扫描线。
        

## 2. 信息架构与页面关系

应用在移动端或桌面端容器中采用**双层视图结构**：

```
                                      [ 应用外壳 App Shell ]
                                                |
                        +-----------------------+-----------------------+
                        |                                               |
                 [ 标签页 1: 首页 Chat ]                        [ 标签页 2: 联系人 Contacts ]
                        |                                               |
             +----------+----------+                        +----------+----------+
             |                     |                        |                     |
       [单角色私聊]           [多角色群聊]              [角色 NPC 列表]         [群聊列表/创建]
             |                     |                        |                     |
      (展示左侧角色面板)     (展示群成员列表及          (点击直接进入          (点击可弹出创建面板 /
                              AI 自动回复状态)            对应的单人私聊)         点击直接进入群聊)
```

## 3. 页面详细规格说明

### 页面 1：首页 Chat（默认首页）

- **顶部栏（Header）**：
    
    - 左侧：闪烁的科技感 `Memoria` Logo 及其版本/状态指示器。
        
    - 中间：极简 HUD 风格搜索框。
        
- **聊天列表项（单聊 vs 群聊列表样式）**：
    
    - **单人私聊项**：
        
        - 左侧：角色头像，其边框会根据角色当前情绪动态发光（如：绿色代表开心，红色代表愤怒，蓝色代表忧郁）。
            
        - 中间：角色名称、当前情绪 Emoji（如 `😊 开心`）以及最新一条消息的缩略文本。
            
        - 右侧：最后消息时间戳及未读消息数红点。
            
    - **多人群聊项**：
        
        - 左侧：三头像重叠堆叠效果。
            
        - 中间：群聊名称（例如：“妖怪调查小组”）+ 最新发送者姓名 + 消息缩略。
            
        - 右侧：时间戳。
            
    - **交互**：鼠标悬停（Hover）时显示快捷操作（置顶、标记已读、删除），并带有数字故障（Glitch）的 hover 特效。
        

### 页面 2：联系人 Contacts 与群组管理

- **NPC 角色名册**：
    
    - 列出所有已解锁的 AI 角色。
        
        
    - 点击 NPC 卡片，直接切换并进入 **单角色聊天页面**。
        
- **群聊管理区**：
    
    - “**+ 创建群聊**” 的交互式快捷卡片。
        
    - **交互弹窗（GroupMemberSelector）**：
        
        1. 允许通过复选框勾选多个 NPC 角色。
            
        2. 设置群聊名称。
            
        3. 实时预览群组头像的拼接或堆叠效果。
            

### 页面 3：单角色聊天页面（角色信息面板 + 聊天区）

- **整体布局（宽屏双栏，移动端可抽屉式收起）**：
    
    - **左侧：角色信息面板 (宽度: 320px-380px)**
        
        - **头像与基础信息**：高清头像。显示姓名、状态标签（如 `善良`、`好奇`、`傲娇`）。
            
        - **RPG 属性状态（关系与情绪）**：
            
            - _好感度进度条_：极具科技感的赛博绿进度条（如 `██████░░ 72%`）。
                
            - _信任度趋势_：当信任值增加时，面板上会浮现向上的绿色微缩箭头。
                
            - _当前情绪_：超大表情符号 + 文本状态（例如：`😊 开心`）。
                
            
    - **右侧：聊天区域 (弹性宽度)**
        
        - **顶部 Header**：角色名称、在线状态（“在线”、“正在思考...”）。
            
        - **消息流（Message Stream）**：动态气泡列表（用户消息在右侧，角色在左侧）。
            
        - **底部输入栏**：未来感控制台。包含：输入框、以及发送按钮。
            

### 页面 4：群聊页面

- **整体布局**：
    
    - 与单聊类似，但左侧面板变更为**群成员状态面板**，展示所有参与聊天的 NPC 及其当前的活跃程度、情绪状态。
        
- **群聊特有交互**：
    
    - 消息气泡上清晰显示对应的角色头像和姓名。
        
    - **发言决策动画**：当用户发送一条指令或问题（如“你们怎么看？”）时，系统界面应展示一个视觉选择器，表现 AI 们在进行“抢答决策”或“连续自动回复”的思考过程。
        

## 4. 代码实现与 UI 生成质量要求

1. **完整的原型功能**：请输出代码干净、功能完整的 React 网页。
    
2. **拒绝原生弹窗**：禁止使用 `alert()` 或浏览器原生弹窗。
    
3. **高动态模拟交互**：
    
    - 允许用户在输入框中输入消息并发送。
